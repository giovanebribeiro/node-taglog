taglog

A tool to generate git tags with messages based on git commits since last tag

Installation

$ npm install --save-dev taglog

Configuration and usage

The recommended usage for this package is with npm-version with --no-git-tag-version flag, because our goal is replace the git tag creation of npm-version and use ours. To do so, we add some configurations in package.json file:

```json

{ ... "taglog": { "lineFormat": "* %h %s", // the line format for 'git-log --pretty'. This is the default option "tagPrefix": "v", // the tag prefix. This is the default option. "tagTitle": "Custom release version title %s" // some custom title for the tag message. '%s' is replaced by the new tag name. // The default option is: 'Release version ' }, "scripts": { "build": "npm version --no-git-tag-version ", "version": "git commit -am \"build: release version\" && taglog $npmpackageversion", "postversion": "git push --follow-tags" }, }

```

And, to bump the version, create the tag and push, use: npm run build -- patch|minor|major

To other infos: taglog --help
